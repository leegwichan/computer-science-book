## 네트워크 계층

### 네트워크 계층 소개
- 네트워크 계층 : 통신을 빠르게 주고받기 위해 패킷이 이동할 최적의 경로를 결정하는 라우팅이 이뤄집니다.
- IP(Internet Protocol) : 네트워크 계층에 핵심 프로토콜, IP 주소 지정과 IP 단편화가 이루어짐
  - IP 주소 지정 : IP 주소를 바탕으로 송수신 대상을 지정하는 것
  - IP 단편화 : 전송하고자 하는 패킷의 크기가 MTU(Maximun Transmission Unit, 최대 전송 단위)보다 클 경우, 이를 MTU 크기 이하의 복수의 패킷으로 나누는 것
  - IP 단편화를 피하는 방법 : PMTUD(Path MTU Discovery, 경로 상의 MTU를 발견하여 단편화를 최소화하는 기술)
- IP 주소 : 네트워크상의 호스트 식발자
  - IPv4, IPv6가 있고, 각각 주소의 길이와 헤더의 구성이 다름
  - IPv4 : IP 주소를 4바이트로 표현함 (1byte * 4개) (ex. 192.168.1.1)
  - IPv6 : IP 주소를 16바이트로 표현함 (2byte * 8개) (ex. 2001:0230:abcd:ffff:0000:0000:ffff:1111)
- ARP(Address Resolution Protocol) : 동일 네트워크 내에 있는 IP 주소를 통해 MAC 주소를 알아내는 프로토콜
  - ARP 요청: 네트워크 전체에 브로드캐스트 메시지를 보내 해당 IP를 가진 호스트를 찾습니다.
  - ARP 응답: 요청받은 IP 주소를 가진 호스트가 자신의 MAC 주소를 담아 송신자에게 일대일(유니캐스트)로 답변을 보냅니다.
  - ARP 테이블 갱신: 수신한 MAC 주소를 메모리(ARP 캐시 테이블)에 저장하여, 동일한 IP로 데이터를 보낼 때 즉시 전송합니다.

### IP 주소의 구조와 할당 체계
- IP 주소의 구성: 네트워크 주소 + 호스트 주소 (둘을 구분하는 경계는 유동적임)
- 클래스풀(Classful) 주소 체계:
  - 정해진 클래스(A~C)를 기준으로 네트워크 주소와 호스트 주소를 나누는 전통적인 방식
  - 네트워크 규모(할당 가능한 호스트 수)에 따라 클래스가 결정
  - 특수 목적 주소: 호스트 주소 부분이 모두 0이면 '네트워크 자체를 의미하는 주소', 모두 1이면 '브로드캐스트 주소'로 사용
- 클래스리스(Classless) 주소 체계
  - 클래스에 얽매이지 않고 서브넷 마스크를 활용해 주소를 더 유연하게 나누는 현대적인 방식
  - 서브넷 마스크: 네트워크 부분은 `1`, 호스트 부분은 `0`으로 채워진 비트열
    - `IP 주소 AND 서브넷 마스크` 비트 연산을 수행하면 해당 호스트가 속한 네트워크 주소를 구할 수 있다
  - CIDR 표기법: `IP 주소 / 서브넷 마스크 내 1의 개수` 형태로 간단히 표기하는 방법

- 공인 IP 주소 (Public IP): 인터넷상에서 통신하기 위해 전 세계에서 유일하게 보장되는 고유한 IP 주소
- 사설 IP 주소 (Private IP): 가정, 회사 등의 내부 네트워크망에서만 사용하기 위해 예약된 IP 주소
    - 인터넷 통신용이 아니며, 주로 라우터(공유기)가 내부 기기들에게 할당
- NAT (Network Address Translation): 내부 네트워크의 사설 IP와 외부 인터넷 통신을 위한 공인 IP를 상호 변환해 주는 기술

- 정적(Static) IP 주소: 사용자가 호스트 기기에 수동으로 직접 입력하여 고정해 둔 주소
- 동적(Dynamic) IP 주소: 네트워크에 연결될 때 시스템으로부터 자동으로 부여받는 주소
- DHCP (Dynamic Host Configuration Protocol): 동적 IP 주소를 자동으로 할당하고 관리해 주는 대표적인 프로토콜
- DHCP의 4단계 동작 과정 (DORA)
  1.  DHCP Discover: 클라이언트가 자신에게 IP를 할당해 줄 DHCP 서버를 찾기 위해 네트워크 전체에 브로드캐스트 메시지를 보냄
  2.  DHCP Offer: 메시지를 받은 DHCP 서버가 클라이언트에게 할당해 줄 IP 주소와 네트워크 정보를 제안
  3.  DHCP Request: 제안을 받은 클라이언트가 해당 IP 주소를 사용하겠다는 확정 요청 메시지를 브로드캐스트로 전송
  4.  DHCP ACK: DHCP 서버가 클라이언트의 요청을 최종적으로 승인하며 IP 할당을 완료

### 라우팅
- 라우팅 : 패킷이 목적지까지 도달하기 위해 거쳐야 할 최적의 경로를 설정하고, 해당 경로로 패킷을 이동시키는 일련의 과정
- 라우터 : 서로 다른 네트워크를 연결하고, 수신된 패킷을 분석해 목적지로 가는 가장 좋은 길을 찾아 전달(포워딩)해 주는 네트워크 장비
- 라우팅 테이블(routing table) : 특정 수신지까지 도달하기 위한 경로 정보 테이블
  - 구성 요소: 수신지 IP 주소, 서브넷 마스크, 게이트웨이, 인터페이스, 메트릭(경로의 비용이나 거리)
  - 디폴트 라우트(Default Route): 라우팅 테이블에 목적지 정보가 없을 때, 기본적으로 패킷을 내보낼 기본 경로(기본 게이트웨이 방향)를 의미

- 정적 라우팅(Static Routing): 목적지에 대한 경로를 수동으로 직접 입력하여 라우팅 테이블을 채우는 방식
- 동적 라우팅(Dynamic Routing): 라우터들이 라우팅 프로토콜을 이용해 서로 정보를 교환하며, 변화하는 네트워크 상황에 맞춰 자동으로 라우팅 테이블을 업데이트하는 방식

- (동적) 라우팅 프로토콜
  - IGP(Interior Gateway Protocol) : 단일 AS 내부에서 정보 교환 사용
    - RIP(Routing Information Protocol) : 거리 벡터 기반의 프로토콜. 목적지까지 거쳐 가는 라우터의 개수(홉 수)를 기준으로 최적의 경로를 판단하며, 인접한 라우터끼리 주기적으로 정보를 교환합니다.
    - OSPF(Open Shortest Path First) : 링크 상태 기반의 프로토콜. 네트워크 전체의 지도를 그리는 LSDB(링크 상태 DB)를 구축한 뒤, 회선의 대역폭(속도)을 바탕으로 메트릭을 계산하여 가장 빠르고 효율적인 경로를 선택
  - EFP(Exterior Gateway Protocol)
    - BGP(Border Gateway Protocol) : 인터넷을 구성하는 핵심 프로토콜로 AS 간의 라우팅에 사용. 단순히 거리나 속도만 보는 것이 아니라, 관리자의 보안 정책과 다양한 속성을 복합적으로 고려하여 경로를 결정

### 네트워크 계층의 한계
- 비신뢰성 프로토콜(unrealiable protocol) : IP 프로토콜이 패킷이 수신지까지 제대로 전송되었다는 보장을 하지 않는 특징
- 비연결형 프로토콜(connectionless protocol) : 송수신 호스트 간에 사전 연결 수립 작업을 거치지 않음
- ICMP(Internet Control Message Protocol) : IP 패킷의 전송 과정에 대한 피드백 메시지를 얻기 위해 사용하는 네트워크 계층 프로토콜
  - ICMP 메시지 종류 : 전송 과정에서 발생한 문제 상황에 대한 오류 보고, 네트워크에 대한 진단 정보
  - 위 특성을 보완하기 위한 도우미 역할을 할 뿐, 신뢰성을 보장하지 않음
