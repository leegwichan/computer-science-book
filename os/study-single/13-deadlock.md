## 교착 상태

### 교착 상태(deadlock) 개요
- 자원 할당 그래프(resource-allocation graph) : 어떤 프로세스가 어떤 자원을 사용/대기 하고 있는지 나타내는 그래프
- **교착 상태 발생 조건**
  - **상호 배제** : 한 프로세스가 사용하는 자원을 다른 프로세스가 사용할 수 없음
  - **점유와 대기** : 어떠한 자원을 할당받은 상태에서 다른 자원을 할당받기를 기다림
  - **비선점** : 특정 자원을 이용하는 프로세스가 끝나야만 다른 프로세스가 이용할 수 있음
  - **원형 대기** : 프로세스들이 원의 형태로 자원을 대기함

### 교착 상태 해결 방법
- 교착 상태 발생 조건 완화
  - 상호 배제 예방 : 시스템 내에 있는 상호 배타적인 모든 자원을 없애버리는 방법 -> 사실상 어려움
  - 점유와 대기 예방 : 프로세스가 자원을 점유한 상태에서 다른 자원을 기다리지 못하게 하는 방법 -> 자원의 활용성이 떨어지고 사실상 일괄 작업 방식으로 동작
  - 비선점 예방 : 모든 자원을 빼앗을 수 있도록 만드는 방법 -> 아사 현상을 일으킴
  - 원형 대기 예방 : 모든 자원에 숫자를 부여하고 숫자가 큰 방향으로만 자원을 할당 -> 프로세스 작업 진행의 유연성이 떨어짐, 자원의 번호 할당 방법 문제

- **교착 상태 회피** : 교착 상태가 발생하지 않을 정도로만 자원을 할당하는 방식
  - 안전 상태 : 교착 상태가 발생하지 않고 모든 프로세스가 안전하게 종료될 수 있는 상태
  - 불안전 상태 : 교착 상태가 발생할 수도 있는 상태
- **은행원 알고리즘** : 대출 금액이 대출 가능한 범위 내이면(안정 상태이면) 허용되지만 그렇지 않으면 거부되는 것과 유사한 방식
    | 사용 변수 | 설명 |
    | :---: | --- |
    | 전체 자원 (Total) | 시스템 내 전체 자원의 수 |
    | 가용 자원 (Available) | 시스템 내 현재 사용할 수 있는 자원의 수 (Total - total Allocation) |
    | 최대 자원 (Max) | 각 프로세스가 선언한 최대 자원의 수 |
    | 할당 자원 (Allocation) | 각 프로세스에 현재 할당된 자원의 수 |
    | 기대 자원 (Expect) | 각 프로세스가 앞으로 사용할 자원의 수 (Max - Allocation) |

- 교착 상태 검출 & 회복 : 교착 상태 발생 여부를 주기적으로 검사하고, 그때마다 회복하는 방식
  - 검출 방법
    - 타임 아웃 : 일정 시간 동안 작업이 진행되지 않은 프로세스를 교착 상태가 발생한 것으로 간주
    - 대기 그래프 분석 : 자원 할당 그래프를 그려 원형 대기인지 파악 (MySQL)
  - 회복 방법
    - 선점 : 교착 상태가 해결될 때까지 한 프로세스 씩 자원을 몰아주는 방법
    - 프로세스 강제 종료 : 데드락과 연관된 프로세스 하나를 강제 종료
