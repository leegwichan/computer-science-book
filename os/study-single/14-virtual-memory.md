## 가상 메모리

### 연속 메모리 할당
- 스와핑(swapping) : 메모리에서 사용되지 않는 일부 프로세스를 보조기억장치로 내보내고 실행할 프로세스를 메모리로 들여오는 메모리 관리 기법
- 메모리 할당 방식
  - 최초 적합(first fit) : 최초로 발견한 적재 가능한 빈 공간에 프로세스를 배치하는 방식
  - 최적 적합(best fit) : 프로세스가 적재될 수 있는 공간 중 가장 작은 공간에 배치하는 방식
  - 최악 적합(worst fit) : 프로세스가 적재될 수 있는 공간 중 가장 큰 공간에 배치하는 방식
- 외부 단편화(external fragmentation) : 프로세스를 할당하기 어려울 만큼 작은 메모리 공간들로 인해 메모리가 낭비되는 현상

### 페이징을 통한 가상 메모리 관리
- 페이징 : 물리 주소 공간을 프레임 단위로 자르고 프로세스의 논리 주소 공간을 페이지 단위로 자른 뒤 각 페이지를 프레임에 할당하는 가상 메모리 관리 기법

- 페이지 테이블 : 어떤 페이지가 어떤 프레임에 있는지에 대한 연결(매핑) 정보 관리
  - CPU 내 PTBR(페이지 테이블 베이스 레지스터)는 각 프로세스의 페이지 테이블에 적재된 주소를 가리킴
  - CPU 곁에(MMU 내에) TLB(Translation Lookaside Buffer)라는 페이지 테이블의 캐시 메모리를 두어 메모리 접근 횟수를 줄임
- 페이지 테이블 엔트리 : 페이지 테이블 구성 요소
  - 페이지 번호, 프레임 번호
  - 유효 비트(페이지 접근 가능 여부), 보호 비트(rwx 권한), 접근 비트(CPU 접근 여부), 수정 비트(수정 여부) 등이 담겨 있음

- 페이징에서의 주소 변환
  - CPU -> 페이지 테이블 : 페이지 번호 + 변위
  - 페이지 테이블 -> 메모리 : 프레임 번호 + 변위

- 페이징의 이점
  - 쓰기 시 복사 : `fork()`를 이용하여 자식 프로세스를 만들 때 같은 프레임 번호를 가리키고, 쓰기 작업이 필요할 때 페이지가 복사됨
- 계층적 페이징 : 페이지 테이블을 2단계 이상으로 두어, 반드시 모든 페이지 테이블에 메모리에 올라와있지 않아도 되도록 함

### 페이지 교체와 프레임 할당
- 요구 페이징 : 페이지가 필요할 때만 메모리에 적재하는 기법

- 페이지 교체 알고리즘
  - FIFO 알고리즘 : 처음 메모리에 올라온 페이지를 스왑 영역으로 보내는 방법 → 자주 사용하는 페이지가 옮겨질 수 있음
  - 2차 기회 페이지 교체 알고리즘 : 특정 페이지에 접근하여 페이지 부재 없이 성공할 경우, 해당 페이지를 큐의 맨 뒤로 이동
  - 최적 알고리즘 : 미래의 메모리 접근 패턴을 보고 대상 페이지를 선정하여 스왑 영역을 보냄 → 실제 구현이 어려움
  - LRU 교체 알고리즘 : 가장 오래 전에 사용한 페이지를 스왑 영역으로 옮길 대상으로 선정한다 → 시간 정보를 위한 메모리가 추가로 필요

- 스레싱(thrashing) : 지나치게 빈번한 페이지 교체로 인해 CPU 이용률이 낮아지는 문제
- 프레임 할당 방식 : 프로세스에게 프레임 개수를 할당하는 방식
  - 균등 할당 : 모든 프로세스에 균등하게 프레임을 할당하는 방식
  - 비례 할당 : 프로세스의 크기에 비례하여 프레임을 할당하는 방식
  - 작업 집합 모델 기반 할당 : 작업 집합(프로세스가 일정 시간 동안 참조한 페이지의 집합) 크디만큼 프레임을 할당하는 방법
  - 페이지 폴트율 기반 할당 : 특정 페이지 폴트율을 유지하도록 페이지를 할당하는 방법
