## 파일 시스템

### 파일과 디렉터리
- 파일 : 보조기억장치에 저장된 관련 정보 집합
  - 메타데이터(속성) : 파일 관련 부가 정보
  - 운영체제는 파일 연산을 위한 시스템 호출을 제공 (생성, 삭제, 열기, 닫기, 읽기, ...)

- 디렉터리 : 여러 개의 파일이나 디렉터리를 묶어 관리하는 도구
  - 트리 구조 디렉터리 : 디렉터리는 트리 구조로 관리됨
  - 운영 체제는 디렉터리를 위한 시스템 호출을 제공
- 경로 : 디렉터리를 이용하여 위치를 특정 짓는 정보
  - 절대 경로 : 루트 디렉터리로부터 시작하는 경로
  - 상대 경로 : 현재 디렉터리로부터 시작하는 경로

### 파일 시스템
- 파티셔닝 : 저장 장치의 논리적인 영역을 구획하는 작업
- 포매팅(formatting) : 파일 시스템을 설정하여 어떤 방식으로 파일을 저장하고 관리할 것인지를 결정하고, 새로운 데이터를 쓸 수 있게 하는 작업
  
- 파일 할당 방법 : 파일을 보조기억장치에 할당하는 방법
  - 연속 할당(contiguous allocation) : 연속적인 블록에 파일을 할당하는 방식 -> 외부 단편화를 야기함
  - 연결 할당(linked allocation) : 각 블록 일부에 다음 블록 주소를 저장하여 각 블록이 다음 블록을 가리키는 형태로 할당하는 방식 -> 반드시 첫 블록부터 읽어야 하며, 오류 발생시 블록 접근 불가
  - 색인 할당(indexed allocation) : 파일의 모든 블록 주소를 색인 블록(index block) 하나에 모아 관리하는 방식

- 파일 시스템
  - FAT 파일 시스템 : FAT(파일 할당 테이블)를 이용하는 연결 할당 기반의 파일 시스템
  - 유닉스 파일 시스템 : i-node를 ㅣ용하는 색인 할당 기반의 파일 시스템
