## CPU 성능 향상 기법

### CPU 성능을 위한 용어
- 클럭(clock) : CPU가 1초 동안 동작하는 속도, 클럭이 높은 CPU는 빠르게 동작한다.
- 코어(core) : CPU 내에서 명령어를 실행하는 부품(ALU + 제어장치 + 레지스터)
- 멀티코어 프로세서 : 여러 개의 코어를 포함하는 CPU
- 스레드(thread) : 실행 흐름의 단위
  - 하드웨어적 스레드 : 하나의 코어가 동시에 처리할 수 있는 명령어 단위 (하이퍼스레딩 기술 - ALU 하나에 레지스터 세트 여러개)
  - 소프트웨어적 스레드 : 운영체제나 애플리케이션 입장에서 본 실행 흐름의 최소 단위 (컨텍스트 스위칭)
- 멀티스레드 프로세서 : 하나의 코어로 여러 개의 명령어를 동시에 실행시킬 수 있는 CPU

### 명령어 병렬 처리 기법
- 명령어 파이프라이닝 : 동시에 여러 개의 명령어를 겹쳐 실행하는 기법
- 파이프라인 위험 : 명령어 파이프라이닝을 사용할 때, 특정 상황에서 성능 향상에 실패하는 경우
  - 데이터 위험 : 데이터 의존적인 두 명령어를 동시에 실행하려고 할 때 파이프라이닝이 제대로 작동하지 않는 것
  - 제어 위험 : 특정 명령어(if, ...)에 의해 프로그램 카운터의 갑작스러운 변화가 생길 때 -> 분기 예측(branch prediction)을 통해 일부 해결
  - 구조적 위험(자원 위험) : 서로 다른 명령어가 동시에 CPU 부품을 사용하려 할 때 발생

- 슈퍼스칼라(superscalar) : 여러 개의 명령어 파이프라인을 두는 기법
- 비순차적 명령어 처리 기법(OoOE, Out-of-order Execution) : 파이프라인의 중단을 방지하기 위해 바꿔 실행해도 무방한 명령어를 먼저 실행하여 명령어 파이프라인이 멈추는 것을 방지하는 기법

### CPU 주요 설계 방식
- ISA(Instruction Set Architecture, 명령어 집합) : CPU가 이해할 수 있는 명령어들의 모음
  - 하드웨어가 소프트웨어를 어떻게 이해할지에 대한 약속
  - 명령어 파이프라인, 슈퍼스칼라, 비순차적 명령어 처리를 사용하기 유리하거나 불리한 명령어 집합이 있다.
- CISC(Complex Instruction Set Computer) : 복잡하고 다양한 종류의 가변 길이 명령어 집합을 활용
  - 적은 명령어만으로 프로그램을 동작시킬 수 있으나, 명령어의 크기와 실행 시간이 일정하지 않음
  - 명령어 파이프라이닝이 매우 어려움
- RISC(Reduced Instruction Set Computer) : 단순하고 적은 종류의 고정 길이 명령어 집합을 활용
  - 메모리 접근을 단순화, 최소화 하는 대신 레지스터를 적극적으로 활용함 (일반적으로 범용 레지스터 개수가 더 많음)
  - 하나의 명령어가 1클럭 내외로 실행되며 파이프라이닝에 최적화되어 있음
