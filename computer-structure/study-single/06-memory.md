## 메모리와 캐시 메모리

### RAM의 특징과 종류
- RAM(Random Access Memory)의 특징 : 휘발성 저장 장치
- RAM의 종류
  - DRAM(Dynamic RAM) : 저장된 데이터가 동적으로 변하는 RAM
  - SRAM(Static RAM) : 비휘발성 메모리, DRAM보다 속도가 빠름, 비효율적이어서 주로 캐시 메모리에 사용
  - SDRAM(Synchronous Dynamic RAM) : 클럭 신호가 동기화된 발전된 형태의 RAM, 클럭 타이밍에 맞춰 CPU와 정보를 주고받을 수 있다.
  - DDR SDRAM(Double Data Rate SDRAM) : 최근 가장 흔히 사용되는 RAM, 대역폭을 넓혀 속도를 빠르게 만든 SDRAM

### 메모리의 주소 공간
- 주소 공간의 종류
  - 물리 주소 : 정보가 실제로 저장된 하드웨어상의 주소
  - 논리 주소 : CPU와 실행 중인 프로그램이 사용하는 주소 (프로그램마다 0번지에서 시작됨)
- MMU(Memory Management Unit, 메모리 관리 장치) : 물리 주소와 논리 주소와의 변환을 담당
  - 각 실행 프로그램의 베이스 레지스터 값을 저장하여 `베이스 레지스터 주소 + 논리 주소 = 물리 주소`를 이용하여 변환

- 메모리 보호 기법
  - 한계 레지스터 : 실행 중인 프로그램의 논리 주소의 최대 크기
  - 명령어를 실행 할 때, 논리 주소가 한계 레지스터 이하 값만을 허용하도록 하여 다른 프로그램을 보호함

### 캐시 메모리
- 저장 장치 계층 구조(memory hierarchy)
  - 각기 다른 용량과 성능의 저장 장치들을 계층화하여 표현한 구조
  - CPU와 가까울수록 속도가 빠르고 용량은 작으며 가격이 비싸다 (레지스터 > 캐시 메모리 > 메모리 > 보조기억장치)
- 캐시 메모리(cache memory)
  - CPU와 메모리 사이에 위치한 SRAM 기반의 저장 장치
  - 코어와 가까운 순서대로 L1 캐시, L2 캐시, L3 캐시가 있다.

- 캐시 적중률
  - 캐시 히트(cache hit) : 자주 사용될 것으로 예측한 데이터가 실제로 들어맞아 캐시 메모리 내 데이터가 CPU에서 활용도리 경우
  - 캐시 미스(cache miss) : 예측이 틀려 메모리에서 필요한 데이터를 직접 가져와야 하는 경우
  - 캐시 적중률(cache hit ratio) : `캐시 히트 횟수 / (캐시 히트 횟수 + 캐시 미스 횟수)`
- 참조 지역성의 원리(locality of reference, principle of locality)
  - 시간 지역성(temporal locality) : 최근에 접근했던 메모리 공간에 다시 접근하려는 경향이 있다.
  - 공간 지역성(spatial locality) : 접근한 메모리 공간 근처를 접근하려는 경향이 있다.
