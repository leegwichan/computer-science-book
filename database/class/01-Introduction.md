## 데이터베이스 개요

1. 데이터와 정보
  - 데이터(Data): 관찰이나 측정으로 수집된 사실이나 값
  - 정보(Information): 의사결정을 위해 데이터를 해석하고 상호간의 관계를 통해 의미를 부여한 것

1. 데이터베이스(DB)의 정의와 특성
  - 정의: 한 조직이 공용하기 위해 통합, 저장한 운영 데이터의 집합
  - 4대 특성: 실시간 접근성, 계속적인 변화(CRUD), 동시 공용, 내용에 의한 참조(위치가 아닌 데이터 값으로 검색)

2. 파일 시스템의 문제점과 DBMS의 등장
  - 기존 파일 시스템은 데이터 종속성(구조 변경 시 응용 프로그램도 변경해야 함)과 데이터 중복성(일관성, 무결성 훼손)이라는 치명적인 단점이 있었다.
  - 이를 해결하기 위해 응용 프로그램과 데이터 사이의 중재자 역할을 하는 DBMS(데이터베이스 관리 시스템)가 등장
  - DBMS 3대 필수 기능: 데이터 구조를 명세하는 정의, 데이터를 처리하는 조작, 정확성과 보안을 유지하는 제어 기능입니다.
  - 궁극적 목적: 응용 프로그램에 영향을 주지 않고 데이터 구조를 변경할 수 있는 데이터 독립성 확보

3. 3단계 데이터베이스 구조
데이터 독립성을 구현하기 위해 데이터베이스를 3단계 관점으로 나눕니다.
  - 외부 스키마(External Schema): 개별 사용자나 응용 프로그램 관점의 논리적 구조입니다.
  - 개념 스키마(Conceptual Schema): 조직 전체 관점의 통합된 데이터 구조입니다.
  - 내부 스키마(Internal Schema): 물리적 저장 장치 관점의 실제 저장 구조입니다.

1. 데이터 언어
사용자와 DBMS 간의 통신 수단입니다.
  - DDL(데이터 정의어): DB 스키마 구조를 정의하고 변경
  - DML(데이터 조작어): 데이터를 검색, 삽입, 삭제, 변경. 절차적 언어(What+How)와 비절차적 언어(What, SQL 등)로 나뉨
  - DCL(데이터 제어어): 보안, 무결성, 회복, 병행 수행 등을 제어하며 주로 데이터베이스 관리자(DBA)가 사용
