## 질의어 처리

1. 질의어 처리 과정
  - 파서(Parser) -> 내부 형태 질의문 -> 질의어 최적기(Optimizer) -> 질의문 계획 -> 실행 코드 생성 -> 런타임 데이터베이스 처리 -> 결과 반환.

2. 질의어 최적화(Query Optimization)의 목적
  - 가장 효율적인 쿼리 실행 전략(계획)을 세우는 것
  - 성능 측정의 가장 핵심적인 기준은 **디스크 I/O 횟수 최소화**, 그 외에 중간 결과의 크기, 응답 시간 등을 고려

3. 규칙 기반 최적화(초기 트리를 최적화된 트리로 변환)
  - 질의문을 관계 대수 트리로 표현한 뒤, 성능을 높이기 위해 구조를 변환
  - **가장 적은 투플 수를 만들어내는 Select 연산을 트리의 가장 아래(단말 노드)로 이동시켜 가장 먼저 실행** -> 이후의 조인 등 연산에서 처리해야 할 중간 데이터의 양을 획기적으로 줄일 수 있다.
  - 카티션 프로덕트 다음에 바로 나오는 실렉트 연산은 묶어서 하나의 조인(Join) 연산으로 통합

4. **Join 연산의 구현 알고리즘**
  - **중첩 루프 조인(Nested-Loop Join)**: 이중 For문처럼 두 릴레이션을 반복하여 비교합니다. 투플 단위로 디스크에 접근하는 것보다, 블록 단위(Block-based)로 메모리에 올려서 비교하는 것이 디스크 I/O를 크게 줄여주어 훨씬 효율적
  - **인덱스 중첩 루프 조인(Indexed Nested-Loop Join)**: 내부 릴레이션의 조인 키에 인덱스가 있는 경우 사용하며, 인덱스를 통해 탐색하므로 디스크 접근 횟수를 대폭 줄일 수 있다.
  - **해시 조인(Hash Join)**: 한쪽 테이블의 조인 키를 이용해 메모리에 해시 테이블을 구성한 뒤 조인하는 방식
  - **정렬/합병 조인(Sort-Merge Join)**: 조인할 두 릴레이션을 조인 키 기준으로 먼저 오름차순 정렬한 뒤, 순차적으로 병합하며 조인하는 방식

5. 기타
  - 데이터 탐색 방식과 비용 : 조건에 맞는 데이터를 찾을 때, 테이블 풀 스캔(선형 탐색), 이원 탐색(이진 탐색), B+트리 등의 인덱스 탐색, 해싱 방식이 있으며 각각의 블록 접근 비용이 다르다.
  - 의미적 질의어 최적화(Semantic Query Optimization) : 스키마에 정의된 무결성 제약조건을 이용해 쿼리 실행 전 미리 결과를 예측함
