## 관계형 데이터베이스(Relational Database)

0. 관계형 데이터베이스(RDBMS)를 사용하는 이유
  - **데이터의 무결성 보장** : ACID 트랜잭션을 지원하여 데이터의 일관성 보장
  - **명확한 스키마를 통한 시스템의 안정성** : 사전에 정의된 엄격한 스키마를 사용하여 데이터를 원하는 형태로 적재

1. 데이터 모델과 관계형 모델의 기초
  - 데이터 모델: 물리적, 추상적으로 존재하는 현실 세계를 단순하고 정형화된 형태로 표현하는 추상화 도구
  - 관계 데이터 모델: 1970년 E. F. Codd가 제안했으며 수학의 릴레이션(Relation)과 집합(Set) 이론에 기초
  - 핵심 용어 매칭:
      - 릴레이션(Relation) = 테이블(Table) = 파일(File)
      - 투플(Tuple) = 행(Row) = 레코드(Record)
      - 애트리뷰트(Attribute) = 열(Column) = 필드(Field)
  - 도메인(Domain): 하나의 애트리뷰트가 취할 수 있는 값들의 집합입니다.

2. 릴레이션의 구조와 특성
  - 릴레이션 스키마(내포): 릴레이션 이름과 애트리뷰트들의 집합입니다. 시간에 따라 변하지 않는 정적 성질을 가집니다. 애트리뷰트의 개수를 차수라고 합니다.
  - 릴레이션 인스턴스(외연): 특정 시점에 릴레이션이 포함하고 있는 투플들의 집합입니다. 삽입, 삭제, 갱신으로 인해 시간에 따라 변하는 동적 성질을 가지며, 투플의 총 개수를 카디널리티(Cardinality)라고 합니다.
  - 릴레이션의 4대 특성:
    1.  투플의 유일성: 중복된 레코드가 존재하지 않습니다.
    2.  투플의 무순서성: 투플 간에는 순서가 존재하지 않습니다.
    3.  애트리뷰트의 무순서성: 애트리뷰트 간에도 순서가 없습니다.
    4.  애트리뷰트의 원자성: 모든 애트리뷰트 값은 논리적으로 더 이상 분해할 수 없는 원자 값(Atomic value)이어야 합니다(정보 부재를 뜻하는 널(Null) 값도 원자 값으로 취급).

3. 키(Key)의 종류 : 각 투플을 유일하게 식별할 수 있는 애트리뷰트의 집합
  - 슈퍼 키(Super Key): 유일성은 만족하지만, 불필요한 속성까지 포함될 수 있어 최소성은 만족하지 않는 키
  - 후보 키(Candidate Key): 각 투플을 식별하는 데 꼭 필요한 애트리뷰트만 포함하여 유일성과 최소성을 모두 만족하는 키
  - 기본 키(Primary Key): 여러 후보 키 중에서 데이터베이스 설계자가 지정한 단 하나의 키. 절대 Null 값을 가질 수 없다.
  - 외래 키(Foreign Key): 다른 릴레이션(또는 자기 자신)의 기본 키를 참조하는 애트리뷰트입니다. 피참조 릴레이션에 존재하는 값이거나 널 값만 가질 수 있습니다.

4. 무결성 제약(Integrity Constraints) : 데이터베이스 상태가 변하더라도 DBMS가 항상 유지해야 하는 필수 제약 조건
  - 개체 무결성(Entity Integrity): 기본 키 값은 어떠한 경우에도 널(Null) 값을 가질 수 없다
  - 참조 무결성(Referential Integrity): 외래 키 값은 반드시 참조하는 릴레이션의 기본 키 값과 일치하거나 널(Null)이어야 하며, 참조할 수 없는 값을 가져서는 안 된다.
