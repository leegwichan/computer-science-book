## 정규화

1. 이상 현상(Anomaly) : 하나의 릴레이션에 여러 종속관계가 섞여 있어 데이터 변경 시 발생하는 부작용입니다.
  - **삽입 이상**(Insertion Anomaly): 원하는 데이터를 추가하기 위해 원하지 않는 불필요한 정보까지 강제로 삽입해야 하는 현상
  - **삭제 이상**(Deletion Anomaly): 특정 데이터를 삭제할 때, 유지되어야 하는 다른 유용한 정보까지 연쇄적으로 함께 손실되는 현상
  - **갱신 이상**(Update Anomaly): 중복 저장된 데이터 중 일부만 갱신되어 데이터 간의 모순이 발생하는 현상
  - 해결책(정규화): 애트리뷰트들 간의 종속 관계를 분해하여, 하나의 종속성은 하나의 릴레이션으로만 표현되도록 구조를 개선하는 것입니다.

2. 함수 종속(Functional Dependency, FD)
  - 정의: 애트리뷰트 X의 값 각각에 대해 애트리뷰트 Y의 값이 최대 하나만 연관될 때 "Y는 X에 함수 종속된다(X → Y)"고 합니다. 이때 X를 결정자, Y를 종속자라고 부릅니다.
  - 완전 함수 종속: 복합 키(2개 이상의 속성으로 구성된 키) 전체에 종속되며, 키의 일부분에는 종속되지 않는 상태
  - 부분 함수 종속: 복합 키의 전체가 아닌 일부 속성만으로도 종속자를 결정할 수 있는 상태
  - 이행적 함수 종속: A → B 이고 B → C 가 성립할 때, A → C 가 성립하는 관계

3. 정규형의 단계 : 정규화는 정보 표현의 무손실, 데이터 중복 최소화, 독립된 관계의 분리를 원칙으로 함. 정규형은 아래로 갈수록 더 엄격한 조건을 만족
  - 제1정규형(1NF): 릴레이션의 모든 도메인이 더 이상 **분해할 수 없는 원자 값**만 가져야 합니다.
  - 제2정규형(2NF): 1NF를 만족하면서, 키가 아닌 모든 애트리뷰트들이 **기본 키에 완전 함수 종속**되어야 합니다(부분 함수 종속 제거)
    - 수강(학생PK, 과목PK, 성적, 지도교수, 학생학과) -> 수강(학생PK, 과목PK, 성적)
  - 제3정규형(3NF): 2NF를 만족하면서, 키가 아닌 모든 애트리뷰트들이 **기본 키에 이행적 함수 종속이 되지 않아야** 합니다.
    - 학생(학생PK, 지도교수PK, 학과) -> 학생(학생PK, 지도교수PK) & 교수(교수PK, 학과)
  - BCNF(Boyce/Codd Normal Form): 강한 3NF라고도 불리며, **릴레이션의 모든 결정자가 후보 키여야 만족**합니다.
    - 배달(가게PK, 배달지역PK, 배달대행지점PK) -> 가게계약(가게PK, 배달 지역PK) & 배달지점(배달지역PK, 배달대행지점PK)
